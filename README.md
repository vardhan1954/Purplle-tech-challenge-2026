OrbiSight Store Intelligence
Overview
OrbiSight is a store intelligence prototype that converts CCTV video into structured retail analytics. It combines YOLOv8 person detection, ByteTrack multi-object tracking, event generation, and FastAPI-based analytics to compute KPIs such as entries, exits, occupancy, funnel conversion, and anomaly signals.

Architecture
The pipeline is intentionally modular and event-driven:

Video Frame Input
YOLOv8 Person Detection
ByteTrack Multi-Object Tracking
Event Generation (app.services.event_generator)
JSON Event Storage (data/events.json)
Metrics Computation (app.services.metrics)
API Exposure (app.main)

Video Input
    ↓
YOLOv8 Person Detection
    ↓
ByteTrack Tracking
    ↓
Event Generator
    ↓
JSON Event Store
    ↓
Metrics Engine
    ↓
FastAPI Endpoints
    ↓
Dashboard / Consumers

Key Components
app/main.py: FastAPI application exposing service endpoints.
app/pipeline/processor.py: Video processing pipeline using YOLOv8 and ByteTrack.
app/services/event_generator.py: Persists structured events to data/events.json.
app/services/metrics.py: Loads events and computes business metrics.
app/models/events.py: Event schema definition.
data/events.json: Event storage file generated during processing.

Features
Real-time person entry detection from video
Unique visitor counting using track IDs
Event-based analytics for store metrics
API endpoints for metrics, funnel, occupancy, anomalies, and aisles
Docker-ready deployment
API Endpoints
GET / - Service status
GET /process - Process data/sample.mp4 and generate events
GET /metrics - Return current KPI metrics
GET /funnel - Return funnel analytics
GET /anomalies - Return anomaly alerts
GET /occupancy - Return current occupancy value
GET /aisles - Return placeholder aisle analytics
Event Model
Events are stored in JSON and contain:

event_id
person_id
session_id
event_type
timestamp
Example event:
{
  "event_id": "...",
  "person_id": "track_17",
  "session_id": "session_17",
  "event_type": "entry",
  "timestamp": "2026-06-02T12:00:00.000000"
}
Installation
Prerequisites
Python 3.11+
pip
Optional: Docker / Docker Compose for containerized deployment
Local Setup
cd d:/OrbiSight
python -m venv .venv
.venv\Scripts\activate
pip install --upgrade pip
pip install -r requirements.txt
Run Locally
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
Then open http://localhost:8000.

Docker
Build and run the container:

docker build -t orbisight-store-intelligence .
docker run -p 8000:8000 orbisight-store-intelligence
Or using Docker Compose:

docker compose up --build
Usage
Place a sample video file at data/sample.mp4.
Start the API service.
Call GET /process to analyze the video and generate events.
Call GET /metrics, GET /funnel, and GET /anomalies to inspect analytics.

Design Decisions
YOLOv8 for fast and accurate person detection
ByteTrack for multi-object tracking and reusable visitor IDs
Event-based analytics to decouple detection from KPI generation
FastAPI for lightweight API delivery and documentation
JSON event store for simple persistence and inspection
Known Limitations
No cross-camera re-identification
Only entry events are generated by the current processing pipeline
aisles endpoint uses placeholder data
Event storage is file-based; no database backend is currently configured
Future Improvements
Add exit, shelf_visit, and purchase event generators
Replace JSON storage with a database layer
Add real-time streaming ingestion and Kafka support
Add multi-camera support and zone analytics
Enhance anomaly detection and dashboard capabilities
Notes
app/api/routes.py is currently unused by the application; routes are defined in app/main.py.
Ensure yolov8s.pt and yolov8n.pt are available in the project root or update model paths.
data/events.json is created automatically when events are generated.
